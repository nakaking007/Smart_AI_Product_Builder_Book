# บทที่ 1: เริ่มต้นเป็น Smart AI Product Builder

> **อย่าเริ่มจากคำถามว่า “จะใช้ AI ตัวไหนดี” ให้เริ่มจากคำถามว่า “ปัญหาอะไรที่ควรแก้”**

## Pain Point

ผู้เริ่มต้นจำนวนมากเปิดเครื่องมือ AI แล้วพิมพ์คำสั่งสั้น ๆ เช่น “ช่วยทำงานให้หน่อย” หรือ “ช่วยสร้างระบบ” ผลลัพธ์จึงไม่ตรงความต้องการ ต้องแก้หลายรอบ และไม่รู้ว่าควรตรวจสอบอะไร

ปัญหาไม่ได้เกิดจาก AI เพียงอย่างเดียว แต่เกิดจากการเริ่มต้นโดยยังไม่กำหนดผู้ใช้ ปัญหา ข้อมูล ผลลัพธ์ และเกณฑ์คุณภาพ บทนี้จะพาคุณสร้างผลงานชิ้นแรกด้วยกระบวนการที่นำกลับมาใช้ได้กับหลายเครื่องมือ

> **Opening Hero Placeholder — ภาพที่ 1-1**
> **ชื่อภาพ:** จาก Pain Point สู่ AI Product ชิ้นแรก
> **องค์ประกอบ:** ผู้ใช้ → ปัญหา → Prompt → ผลลัพธ์ → การตรวจสอบ
> **Aspect Ratio:** 16:9
> **Callout:** เน้นว่าต้องเริ่มจากปัญหา
> **Alt Text:** แผนภาพแสดงลำดับการสร้าง AI Product ตั้งแต่ค้นหาปัญหาจนตรวจสอบผลลัพธ์

## เป้าหมายบทเรียน

เมื่อจบบทนี้ คุณจะสามารถ

- แยกความแตกต่างระหว่างการทดลองใช้ AI กับการสร้าง AI Product
- เลือก Pain Point ที่เหมาะกับโครงงานแรก
- สร้าง Prompt แบบมีโครงสร้าง
- ทดสอบและปรับปรุงผลลัพธ์ด้วย Checklist
- สร้าง Mini Project ผู้ช่วยสรุปรายงานประชุม
- บันทึก Prompt และหลักฐานการทดสอบไว้ใช้ซ้ำ

## สิ่งที่ต้องเตรียม

- คอมพิวเตอร์ แท็บเล็ต หรือโทรศัพท์ที่เชื่อมต่ออินเทอร์เน็ต
- บัญชีเครื่องมือ AI สนทนาที่คุณได้รับอนุญาตให้ใช้
- โปรแกรมจดบันทึกหรือไฟล์ Markdown
- เวลา 45–60 นาทีสำหรับบทเรียน
- ข้อมูลตัวอย่างที่ไม่มีข้อมูลลับหรือข้อมูลส่วนบุคคลจริง

> **คำเตือนด้านข้อมูล:** แบบฝึกหัดในบทนี้ใช้ข้อมูลสมมติ ห้ามนำรายงานประชุมจริงที่มีชื่อ เบอร์โทร อีเมล ข้อมูลงบประมาณที่ยังไม่เปิดเผย หรือข้อมูลส่วนบุคคลเข้าสู่บริการ AI โดยไม่ได้รับอนุญาต

| --- | --- | --- | --- |
| New Chat | เริ่มบทสนทนาใหม่ | เมื่อเปลี่ยนงานหรือเปลี่ยนบริบท | บทสนทนาใหม่อาจไม่รู้ข้อมูลจากงานเดิม |
| Model Selector | เลือกโมเดลหรือโหมด | เมื่องานต้องการความเร็ว คุณภาพ หรือความสามารถต่างกัน | บางบัญชีไม่มีเมนูนี้และราคาอาจต่างกัน |

| Settings / Privacy | จัดการภาษา ข้อมูล และความเป็นส่วนตัว | ก่อนใช้งานกับงานองค์กร | ตัวเลือกแตกต่างกันตามบริการและบัญชี |

## Step by Step

### Step 1: เลือก Pain Point ที่เกิดซ้ำ

> **Screenshot Placeholder — ภาพที่ 1-3**
> แบบฟอร์มรายการงานซ้ำ 3 งาน พร้อมกรอบเน้นงานที่เลือก
> **Alt Text:** ตัวอย่างตารางคัดเลือก Pain Point สำหรับโครงงานแรก

เริ่มจากงานที่คุณทำเป็นประจำและใช้เวลามาก เขียนรายการอย่างน้อย 3 งาน เช่น

- สรุปรายงานประชุม
- ร่างอีเมลติดตามงาน
- จัดหัวข้อแผนการสอน
- แปลงบันทึกเป็น Checklist
- ตรวจรูปแบบเอกสาร

เลือกเพียงหนึ่งงานสำหรับบทนี้ โดยใช้เกณฑ์ต่อไปนี้

1. เกิดซ้ำอย่างน้อยเดือนละ 2 ครั้ง
2. มี Input และ Output ชัดเจน
3. ผลลัพธ์สามารถให้มนุษย์ตรวจได้
4. ไม่ต้องใช้ข้อมูลลับในการฝึกครั้งแรก
5. หากทำได้เร็วขึ้นจะเกิดประโยชน์จริง

**งานตัวอย่างของบทนี้:** สรุปรายงานประชุมจากบันทึกสมมติ

### Step 3: เตรียมข้อมูลสมมติ

> **Screenshot Placeholder — ภาพที่ 1-6**
> เปรียบเทียบข้อมูลก่อนและหลังลบข้อมูลส่วนบุคคล โดยใช้ข้อมูลสมมติ
> **Alt Text:** ตัวอย่างการเตรียมข้อมูลที่ปลอดภัยก่อนส่งให้ AI

ใช้บันทึกต่อไปนี้เพื่อฝึกโดยไม่เสี่ยงต่อข้อมูลจริง

~~~text
การประชุมทีมโครงการ วันที่ 10 สิงหาคม เวลา 09.00 น.

ผู้เข้าร่วม: ผู้จัดการโครงการ เจ้าหน้าที่เนื้อหา เจ้าหน้าที่เว็บไซต์

ประเด็น:
- ต้นฉบับบทที่ 1 พร้อมตรวจรอบแรก
- ภาพประกอบยังขาด 3 ภาพ
- เว็บไซต์ตัวอย่างต้องทดสอบบนโทรศัพท์
- ตกลงให้เจ้าหน้าที่เนื้อหาตรวจภาษา
- เจ้าหน้าที่เว็บไซต์ทดสอบหน้าจอภายในวันศุกร์
- ยังไม่ได้กำหนดผู้รับผิดชอบภาพประกอบ
~~~

ก่อนส่งข้อมูล ให้ตรวจว่าไม่มีข้อมูลต่อไปนี้

- รหัสผ่านหรือ Token
- เลขประจำตัวประชาชน
- เบอร์โทรศัพท์หรืออีเมลส่วนตัว
- ข้อมูลสุขภาพ
- ข้อมูลงบประมาณหรือสัญญาที่ไม่เปิดเผย
- ชื่อบุคคลจริงที่ไม่จำเป็น

Prompt ที่ดีควรบอกบทบาท งาน บริบท Input กฎ รูปแบบผลลัพธ์ และการตรวจคุณภาพ ใช้ Prompt Box ต่อไปนี้เป็นต้นแบบ

## Prompt พร้อมใช้

### Meeting Action Summary

- **เครื่องมือ:** AI Chat ที่รองรับภาษาไทย
- **ระดับ:** Beginner
- **วัตถุประสงค์:** แปลงบันทึกประชุมเป็นสรุปและตารางติดตามงาน

~~~text
Role
คุณคือผู้ช่วยเลขานุการโครงการที่เขียนภาษาไทยกระชับและไม่สร้างข้อมูลขึ้นเอง

Task
วิเคราะห์บันทึกการประชุมและจัดทำสรุปสำหรับติดตามงาน

Context
ผู้ใช้ต้องการทราบประเด็นสำคัญ มติ งานที่ต้องทำ ผู้รับผิดชอบ และกำหนดเวลา

Input
[วางบันทึกการประชุมที่ผ่านการลบข้อมูลส่วนบุคคลแล้ว]

Rules
1. ใช้เฉพาะข้อมูลใน Input
2. ห้ามเดาชื่อผู้รับผิดชอบหรือกำหนดเวลา
3. หากข้อมูลไม่มี ให้เขียนว่า “ยังไม่ระบุ”
4. แยกข้อเท็จจริงออกจากข้อเสนอแนะ
5. ใช้ภาษาไทยอ่านง่าย

Output
1. สรุปภาพรวมไม่เกิน 5 บรรทัด
2. รายการมติ
3. ตาราง: งาน | ผู้รับผิดชอบ | กำหนดเวลา | สถานะ
4. รายการข้อมูลที่ยังขาด
5. Checklist สำหรับผู้ตรวจรายงาน

Quality Check
ก่อนส่ง ให้ตรวจว่าไม่มีข้อมูลใดที่ไม่ได้อยู่ใน Input
~~~

**ผลลัพธ์ที่คาดหวัง:** ได้สรุปที่แยกมติ งาน และข้อมูลที่ยังขาด โดยไม่เดาผู้รับผิดชอบภาพประกอบ

**วิธีตรวจสอบ:** เปรียบเทียบทุกแถวกับบันทึกต้นฉบับ หากข้อความใดหาแหล่งที่มาไม่ได้ให้ลบหรือแก้ไข

**ข้อควรระวัง:** AI อาจเติมคำที่ดูสมเหตุผลแต่ไม่มีใน Input ผู้ใช้ต้องตรวจทุกครั้ง

> **ชื่อภาพ:** โครงสร้าง Prompt 7 ส่วน

> **Aspect Ratio:** 4:3
> **Alt Text:** แผนภาพกล่องเจ็ดส่วนของ Prompt สำหรับสรุปรายงานประชุม

> **Professional Tip:** ปรับ Prompt ทีละเงื่อนไขและเก็บคะแนนทุก Version เพื่อให้ทราบว่าการเปลี่ยนแปลงใดช่วยเพิ่มคุณภาพจริง

- เก็บทั้ง Prompt และ Test Case

## สรุป

ในบทนี้คุณไม่ได้เพียงถาม AI แต่ได้เริ่มทำงานแบบ AI Product Builder คุณเลือกปัญหา กำหนดผู้ใช้ เตรียมข้อมูลที่ปลอดภัย สร้าง Prompt แบบมีโครงสร้าง ตรวจผลด้วยคะแนน และเก็บผลงานเป็น Version

หลักสำคัญคือเริ่มจาก Pain Point ไม่ใช่ชื่อเครื่องมือ และให้มนุษย์ตรวจผลลัพธ์ก่อนนำไปใช้จริงเสมอ

## แบบฝึกหัด

1. เลือกงานซ้ำอีกหนึ่งงาน แล้วเขียน Input และ Output ให้ชัดเจน
2. เพิ่ม Test Case ที่จงใจไม่ระบุผู้รับผิดชอบ เพื่อดูว่า AI จะเดาหรือไม่
3. เปลี่ยนรูปแบบ Output จากตารางเป็น Checklist แล้วเปรียบเทียบว่าแบบใดเหมาะกับผู้ใช้มากกว่า
4. เขียน Privacy Checklist สำหรับงานของคุณอย่างน้อย 5 ข้อ
5. อธิบายว่าทำไม Prompt ที่ยาวกว่าจึงไม่ได้หมายความว่าดีกว่าเสมอ
