# วิธีใช้หนังสือ

หนังสือเล่มนี้เป็น Workbook และคู่มือปฏิบัติในเล่มเดียว วิธีที่ได้ผลที่สุดคืออ่านสั้น ๆ แล้วทำตามทันที ไม่ควรอ่านรวดเดียวจนจบโดยยังไม่ได้เปิดเครื่องมือ

## วิธีเรียนหนึ่งบท

1. อ่าน Pain Point และกำหนดผลงานปลายบท
2. ตรวจ Prerequisite และบัญชีที่ต้องใช้
3. ทำ Step by Step ตามลำดับ
4. ใช้ Prompt Box โดยแทนค่าข้อความในวงเล็บเหลี่ยม
5. ตรวจผลด้วยเกณฑ์ผ่าน ไม่ตัดสินจากความรู้สึก
6. ทำ Workshop ตามโจทย์กลาง
7. ทำ Mini Project จากงานของตนเอง
8. ตรวจ Checklist และส่งหลักฐาน

> **[INFOGRAPHIC-02-01] ชื่อภาพ: วงจรเรียนหนึ่งบท**
> **จุดประสงค์:** ช่วยให้ผู้เรียนไม่ข้ามการทดสอบ
> **สิ่งที่ต้องปรากฏ:** Read → Do → Test → Improve → Submit
> **คำบรรยายใต้ภาพ:** วงจรการเรียนที่จบด้วยผลงานและหลักฐาน

## วิธีใช้ Prompt

Prompt ในหนังสือเป็นต้นแบบ ไม่ใช่คำวิเศษ ให้ดำเนินการดังนี้

- อ่านวัตถุประสงค์และเครื่องมือที่ระบุ
- เตรียม Input ให้ครบ
- แทนค่า `[ข้อความตัวอย่าง]` ด้วยข้อมูลของตนเอง
- ลบข้อมูลส่วนบุคคล รหัสผ่าน และข้อมูลลับ
- ส่ง Prompt แล้วตรวจ Output กับเกณฑ์
- หากผลไม่ผ่าน ให้แก้ข้อมูลหรือกฎทีละจุด
- บันทึก Prompt ที่ผ่านเป็นเวอร์ชัน เช่น `v1.0`

อย่าคัดลอกผลลัพธ์ไปใช้โดยไม่อ่าน โดยเฉพาะตัวเลข กฎหมาย แหล่งอ้างอิง ชื่อบุคคล และข้อกล่าวอ้างทางธุรกิจ

## วิธีใช้ Workshop

Workshop เป็นกิจกรรมที่ผู้สอนและผู้เรียนทำด้วยโจทย์เดียวกัน ให้เตรียมไฟล์ตามรายการ จับเวลา และส่งผลงานตามเกณฑ์รับ ระหว่าง Workshop ผู้สอนควรช่วยแก้ปัญหากระบวนการ ไม่ทำงานแทนผู้เรียน

หลักฐาน Workshop อาจประกอบด้วยลิงก์ ไฟล์ Markdown, PDF, ภาพหน้าจอ หรือ Test Report ตามที่บทกำหนด

## วิธีใช้ Mini Project

Mini Project ให้ผู้เรียนเปลี่ยนตัวอย่างกลางเป็นปัญหาของตนเอง ภายในเวลาประมาณ 20–30 นาที เลือกขอบเขตเล็กที่เสร็จและทดสอบได้ ดีกว่าเลือกระบบใหญ่แต่ไม่มีผลงานส่ง

ใช้สูตร `ปัญหาเดียว + ผู้ใช้หนึ่งกลุ่ม + ผลลัพธ์หนึ่งชนิด + เกณฑ์ผ่านที่วัดได้`

## วิธีใช้ Screenshot Placeholder

รหัส เช่น `[SCREENSHOT-04-03]` คือจุดวางภาพในขั้น Artwork คำอธิบายใต้ Placeholder ระบุสิ่งที่ต้องจับภาพ จุดที่ใส่ลูกศร และ Caption ผู้เรียนฉบับร่างสามารถใช้คำอธิบายแทนภาพได้ ส่วนฝ่ายผลิตต้องใช้ภาพจากหน้าจอจริงและปิดข้อมูลส่วนบุคคล

## วิธีใช้ Diagram และ Infographic

Diagram ใช้อธิบายความสัมพันธ์หรือ Workflow ไม่ใช่ของตกแต่ง ทุกภาพต้องมีชื่อ จุดประสงค์ สิ่งที่ต้องปรากฏ และคำบรรยาย หากภาพไม่ช่วยให้เข้าใจเร็วขึ้น ควรใช้ข้อความหรือตารางแทน

## วิธีใช้ QR Code

QR Code ใช้เปิดไฟล์ฝึก วิดีโอ หรือตัวอย่างออนไลน์ ต้องมี URL จริง ทดสอบจากโทรศัพท์อย่างน้อยสองเครื่อง และพิมพ์ URL สำรองไว้ใต้ภาพ ห้ามสร้าง QR จากลิงก์ชั่วคราวหรือไฟล์ที่ผู้เรียนไม่มีสิทธิ์เปิด

## วิธีใช้ไฟล์ตัวอย่าง

- คัดลอกไฟล์ต้นฉบับก่อนแก้
- เปลี่ยนชื่อเป็น `ชื่อกิจกรรม_ชื่อผู้เรียน_เวอร์ชัน`
- ใช้ข้อมูลจำลอง
- เก็บ Source, Prompt, Output และ Test Report แยกกัน
- ตรวจสิทธิ์ก่อนแชร์

## วิธีส่งงาน

สร้างโฟลเดอร์หนึ่งชุดต่อผลงาน โดยมี `README.md`, ไฟล์ผลงาน, Prompt/Instructions, Test Report และภาพหลักฐาน ตั้งชื่อไฟล์เป็นภาษาไทยหรืออังกฤษให้สม่ำเสมอ หลีกเลี่ยงชื่อ `final-final2` ให้ใช้เวอร์ชันและวันที่

ตัวอย่าง

```text
AI_Assistant_Project/
├── README.md
├── instructions_v1.1.md
├── test_report.md
├── evidence/
└── deliverable/
```
